# Estrategia de Ramas

El proyecto utiliza Git como sistema de control de versiones.

## Ramas principales
- main: contiene la versión estable del proyecto.
- develop: contiene los cambios en desarrollo.

## Ramas de trabajo
- feature/*: ramas utilizadas para desarrollar tareas específicas.

## Flujo de trabajo
1. Crear una rama feature desde develop.
2. Realizar los cambios.
3. Enviar los cambios mediante Pull Request hacia develop.
4. Revisar y aprobar el Pull Request.
